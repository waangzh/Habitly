# Habitly

Habitly 是一个偏陪伴式、轻提醒风格的微信习惯养成小程序，核心目标不是管理待办事项，而是帮助用户把“想坚持的事”变成可以长期延续的小习惯。

它强调短路径打卡、柔和卡片化视觉、低负担记录和持续反馈，整体体验更接近“日常陪伴型习惯产品”，而不是列表式任务工具。

## 项目概要

本项目包含两个部分：

- **微信小程序前端**：负责首页打卡、项目管理、历史记录、成绩单、成就页、个人资料等核心体验
- **NestJS 后端服务**：负责项目、打卡、历史、报告、成就、用户资料和 AI 能力等接口

前端通过 `services/` 统一访问后端接口，保证页面层聚焦展示与交互，业务逻辑尽量收敛在服务层。

## 产品特色

- **习惯养成导向**：围绕“每天做一点”设计，不把产品做成传统任务清单
- **打卡路径短**：首页直接完成打卡，减少多层确认和跳转
- **轻量补充记录**：打卡后可补充心情、评分、数值等信息
- **鼓励式反馈**：通过报告、成就、趋势等方式强化正向感受
- **柔和卡片化体验**：界面强调留白、圆角卡片、轻色系和清晰层级
- **AI 辅助能力**：支持习惯草案、提醒建议、报告洞察等增强能力

## 项目架构

### 1. 前端结构

```text
habit-to-do/
├─ pages/          # 页面
├─ components/     # 通用组件
├─ services/       # 接口请求与业务数据入口
├─ store/          # 轻量 UI 状态映射
├─ styles/         # 全局主题样式
├─ utils/          # 工具函数
└─ server/         # 后端服务
```

### 2. 小程序页面层

当前主要页面包括：

- `home`：首页，按日期查看并直接打卡
- `projects`：项目列表
- `project-detail`：项目详情、统计与记录
- `project-form`：新建/编辑项目
- `history`：历史记录
- `report-card`：成绩单
- `achievements`：成就页
- `more`：更多与个人资料
- `paused-projects`：暂停项目列表

### 3. 前端服务层

`services/habitService.js` 是核心业务入口，负责：

- 获取项目列表与详情
- 创建、编辑、暂停、删除项目
- 首页数据获取
- 打卡与补充记录
- 历史、成绩单、成就数据获取
- 个人资料读取与更新

页面层主要消费这里暴露的方法，避免把业务判断分散到多个页面中。

### 4. 后端服务层

`server/` 基于 **NestJS + TypeORM** 构建，按模块划分能力，主要包括：

- `auth`：登录、会话、令牌相关能力
- `users`：用户资料
- `projects`：习惯项目管理
- `checkins`：打卡与补充记录
- `home`：首页聚合数据
- `history`：历史记录
- `reports`：成绩单与报告
- `achievements`：成就统计
- `more`：更多页概览
- `ai`：AI 生成与辅助能力

## 关键功能

### 习惯项目管理

- 新建习惯项目
- 编辑项目内容
- 暂停 / 恢复项目
- 删除项目
- 支持图标、口号、颜色主题、提醒时间、周期设置等信息

### 首页打卡

- 按日期查看当日习惯项目
- 直接完成打卡或取消打卡
- 打卡后补充心情、评分、数值记录
- 保持“今天该做什么”一眼清晰

### 数据回顾与激励

- 项目详情统计
- 历史记录按月查看
- 成绩单展示阶段性完成情况
- 成就页展示连续打卡、累计打卡等成果

### 用户与 AI 能力

- 个人资料查看与编辑
- AI 辅助生成习惯草案
- AI 提供打卡引导、反思建议、报告洞察等内容

## 技术栈

### 前端

- 微信小程序原生框架
- WXML / WXSS / JavaScript

### 后端

- NestJS
- TypeORM
- MySQL
- Redis
- Jest

## 快速开始

### 运行小程序前端

1. 使用微信开发者工具打开项目根目录 `habit-to-do`
2. 配置小程序 AppID
3. 在开发者工具中进行编译、预览和调试

### 运行后端服务

在 `server/` 目录下执行：

```bash
npm install
npm run start:dev
```

常用命令：

```bash
npm run build
npm test
npm run migration:run
npm run migration:revert
npm run seed
```

## 项目价值

Habitly 更关注“帮助用户开始”和“帮助用户坚持”，而不是堆叠复杂规则。它试图用更轻、更温和、更有陪伴感的方式，让习惯记录这件事变得可持续。
